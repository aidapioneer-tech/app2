# CLAUDE.md — правила работы с репозиторием

Этот файл читают AI-агенты и разработчики перед началом работы.

## Язык

Общение с AI — на русском. Документация — на русском.

## Ветки и пуши

- **В `main` напрямую не пушим никогда.** Все изменения — только через Pull Request.
- Репозиторий приватный, поэтому PR описывается текстом (заголовок + тело) без внешних ревьюеров.
- Каждый PR — одна логическая задача. Не мешать несвязанные изменения в один PR.

## Процесс валидации PR

- Перед созданием PR убедиться, что `pnpm lint` и `pnpm typecheck` проходят без ошибок (локально всё сразу — `scripts/check.sh` / `scripts/check.ps1`).
- На каждый PR автоматически запускается CI (`.github/workflows/ci.yml`): lint, typecheck, test, build. Мерж — после зелёного CI.
- PR описывается понятно: что сделано и зачем.

## Мерж

**Мерж делается только по явному указанию владельца.** AI не мержит PR по собственной инициативе — только когда владелец явно просит слить. Без такого указания AI останавливается на создании PR.

## База знаний

После каждого смерженного PR — отправлять оповещение через issue в https://github.com/aidapioneer-tech/aida/issues — кратко описать что изменилось в проекте (новые фичи, изменения архитектуры, баги). Это сигнал для обновления документации в `aida/docs`.

## Работа AI-агента

_(практики заимствованы из [b24ui AGENTS.md](https://github.com/bitrix24/b24ui/blob/main/AGENTS.md))_

- **Загружать файлы под задачу, а не всё подряд.** Читать только релевантные задаче файлы/доки. Для больших ревью — точечно, чтобы не падать по таймауту.
- **Skill-файлы** (если появятся) держать в трекаемой `skills/<skill-name>/`, не в `.claude/skills/` (последняя в `.gitignore`). При наличии обоих — источник правды `skills/`.
